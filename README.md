##设计实现

<!-- 客户端通过抢占redis锁的方式来决定自身为服务端还是客服端，成为服务端后要发布gRPC服务，并将本身ip和端口号存储在redis数据库中。其他未抢占到redis锁的以客户端模式启动，连接至服务器。用户可以通过客户端向服务端发送添加任务请求，服务端接受请求后会将任务持久化到数据库中，以防止宕机造成数据丢失，同时方便各客户端查询任务执行情况。 -->
